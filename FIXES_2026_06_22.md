# Исправления бота Nomad Clinic

## Дата: 2026-06-22

### 🐛 Исправленные ошибки

#### 1. Имя пациента не спрашивалось
**Проблема**: После выбора услуги слово "Да" попадало в поле имени пациента.

**Решение**:
- Добавлено состояние `waiting_confirmation` между выбором услуги и сбором данных
- Теперь после выбора услуги бот ждёт явного подтверждения ("да", "хочу записаться")
- Только после подтверждения начинается запрос имени
- Добавлена проверка: имена короче 3 символов отклоняются

**Файл**: `src/services/rag_service.ts`

---

#### 2. Принимались некорректные даты (31 февраля)
**Проблема**: Бот принимал любые даты без проверки на существование.

**Решение**:
- Добавлен метод `validateDate()` для проверки дат
- Проверяется:
  - Количество дней в месяце (31 февраля → ошибка)
  - Диапазон дней (1-31)
  - Диапазон месяцев (1-12)
  - Дата не в прошлом
- При ошибке показывается понятное сообщение

**Примеры**:
```
❌ 31 февраля → "В февраля 2026 года только 28 дней!"
❌ 32 января → "День должен быть от 1 до 31"
❌ 15.13 → "Месяц должен быть от 1 до 12"
✅ 28 февраля → OK
✅ 31 марта → OK
```

**Файл**: `src/services/rag_service.ts`

---

#### 3. Google Sheets не записывал данные
**Проблема**: Запись подтверждалась, но в таблицу не попадала.

**Решение**:
- Добавлено логирование статуса сохранения
- Улучшена обработка ошибок
- Добавлено сообщение пользователю о статусе сохранения

**Файл**: `src/services/rag_service.ts`, `src/services/google_sheets_service.ts`

### ✨ Новые возможности

#### 4. Интеграция с Google Calendar
**Возможности**:
- Проверка реальных доступных слотов через Calendar API
- Автоматическое создание событий при подтверждении записи
- Синхронизация с Google Sheets

**Настройка**:
1. Создать сервисный аккаунт в Google Cloud
2. Включить Google Calendar API
3. Скачать ключ (`google-credentials.json`)
4. Настроить переменные окружения:
   ```bash
   GOOGLE_CALENDAR_ENABLED=true
   GOOGLE_CALENDAR_ID=primary
   ```

**Файлы**:
- `src/services/google_calendar_service.ts` (новый)
- `GOOGLE_CALENDAR_SETUP.md` (инструкция)

### 🔧 Технические детали

#### Валидация дат
```typescript
private validateDate(dateStr: string): { valid: boolean; error?: string }
```

Поддерживаемые форматы:
- `сегодня`, `завтра`, `послезавтра`
- `бүгін`, `ертең` (казахский)
- `25.06`, `25/06`, `25-06`
- `25.06.26`, `25.06.2026`
- `25 июня`, `25 февраля`

#### Google Calendar Service
```typescript
class GoogleCalendarService {
  async isSlotAvailable(date, time, duration): Promise<boolean>
  async getAvailableSlots(date, baseTimes, duration): Promise<string[]>
  async createEvent(booking): Promise<boolean>
  async cancelEvent(eventId): Promise<boolean>
}
```
